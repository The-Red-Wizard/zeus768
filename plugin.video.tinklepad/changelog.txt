v1.9.10 (2026-01-15)
====================
- FIX (Torbox): Magnet/torrent links now resolve correctly. Three blocking bugs were patched in resources/lib/debrid.py:
  • All /torrents/mylist, /webdl/mylist and /usenet/mylist polls now send `bypass_cache=true`. Torbox otherwise serves a ~600s stale cache, so the freshly-added torrent never reported as ready within the 90s polling window — the script silently returned None on every magnet
  • Added a /torrents/checkcached pre-flight using the BTIH hash extracted from the magnet URI. Uncached torrents (which can take hours to download on Torbox's side) now fail fast instead of looping, letting other debrids or scraper fallbacks pick up the slack
  • Extended the "ready" detection to include `download_state` values `cached`, `completed`, `uploading` and `seeding`, matching Torbox's documented states
  • Handle DUPLICATE_ITEM responses from /torrents/createtorrent by re-using the existing torrent_id instead of bailing out
  • seed parameter changed from "3" (no-seed) to "1" (auto), matching the Torbox default
- No changes to RD / AD / PM resolvers
- Bumped to v1.9.10
