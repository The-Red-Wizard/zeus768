Genesis Changelog
==================

Version 9.2.6 (January 2026 - Critical Python 3 Fixes)
- Fixed "a bytes-like object is required, not 'str'" error in urllib2.Request POST data
- Fixed StringIO.StringIO to use BytesIO for handling binary zip data in episodes.py
- Added proper bytes-to-string decoding for TVDB XML data
- Fixed NoneType error in movies.py when IMDB list returns None
- All menus and categories should now load properly

Version 9.2.5 (January 2026 - IMDB/TMDB Menu Fix)
- Fixed critical Python 3 bug: str.decode() called on string objects causing menus to fail
- Fixed _ensure_str() in client.py to properly decode bytes to strings
- Added isinstance(result, bytes) checks before .decode('iso-8859-1') calls in:
  * movies.py (3 locations - imdb_list, imdb_user_list, imdb_person_list)
  * tvshows.py (3 locations - imdb_list, imdb_watchlist, imdb_user_list)  
  * episodes.py (1 location - scn_list)
  * uploadrocket.py (3 locations - resolve function)
- Movie and TV Show menus now load correctly from IMDB

Version 9.2.4 (Kodi 21+ Compatibility Update)
- Fixed Python 3 compatibility issues for Kodi 21 Omega and above
- Fixed base64.b64encode() bytes/string encoding issues
- Fixed xrange compatibility for Python 3
- Fixed yesnoDialog deprecated API for Kodi 19+
- Fixed tvdb_key and fanarttv_key bytes decoding
- Fixed self.lang to self.info_lang in tvshows.py
- Added proper error handling for string/bytes operations
- Updated addon version to 9.2.4
