v2.9.55 (2026-01) - IDRIVE ACCOUNT MODE + SYNC.COM RESOLVER FIXES
- iDrive Cloud Library now supports two real account-level modes
  in addition to the existing shared-link mode:
    * WebDAV (iDriveSync at https://www.idrivesync.com/webdav, or
      any user-supplied WebDAV endpoint) - Basic auth, PROPFIND
      walks the entire video tree, direct play in Kodi.
    * iDrive e2 (S3-compatible object storage) - access key /
      secret, native AWS SigV4 signing, presigned URLs that Kodi
      plays directly. No external libraries required.
- Settings -> Cloud Sources -> "iDrive Account" lets you pick the
  mode and "Test iDrive Connection" verifies your credentials
  without leaving Kodi.
- If the new account fields are left blank, the addon falls back
  transparently to the previous iDrive shared-link behaviour, so
  existing setups keep working untouched.
- Sync.com share-link resolver rewritten: now extracts link_id +
  cachekey from the share page (or URL), calls the public
  cp.sync.com/getlinkmeta endpoint to obtain the signed
  download_url, and falls back to the legacy regex scrape.
  Sync.com remains share-link-only (no public account API exists -
  the service is zero-knowledge encrypted).

v2.9.45 (2026-01) - CLOUD SECTION (GOOGLE DRIVE + MEGA.NZ)

  the file's share/public URL to it. For Google Drive the file is auto-
  shared (anyone-with-link, reader) on first push.
- Playback resolves the cloud file to a streamable URL (Drive: alt=media
  with bearer header; MEGA: temporary HTTPS link) and hands it to Kodi.

v2.9.28 (2026-01) - TORBOX SMOOTHER UNCACHED PLAYBACK
- TorBox: uncached torrents now auto-resolve by default. SALTS
  starts the download on TorBox and launches the player the
  moment TorBox marks the file as ready - no more silent
  "skips" on non-cached sources.
- TorBox: replaced the blocking modal progress dialog with a
  small non-intrusive background progress bar. Kodi stays fully
  usable while TorBox prepares the file.
- TorBox: friendly notifications now tell you exactly what is
  happening - "Not cached - downloading on TorBox, playback will
  start automatically when ready", periodic progress updates,
  and a clear "trying next source" message if it gives up.
- TorBox: poll interval cut from 3s to 1s so playback starts
  almost instantly once TorBox has the file.
- TorBox: max wait for uncached torrents bumped from 5 to 10
  minutes (configurable 1-30) so larger 1080p/4K files have
  time to land on TorBox's CDN.
- Note: TorBox's API does NOT support streaming a torrent
  while it is still being downloaded the way Real-Debrid does
  (their /torrents/requestdl link only becomes valid once the
  file is fully present on the CDN). What this release gives
  you is the closest possible UX to RD's "just plays".

v2.9.26 (2026-01) - DEAD TORRENT FILTER + QR DIALOG REWRITE
- Added: dead-torrent filter. Torrents with 0 (or fewer than
  "Minimum seeders") seeders are dropped from the source list
  before you ever click them, so the debrid resolver doesn't
  hang trying to fetch a magnet nobody is sharing. Cached
  debrid torrents are exempt (the debrid server already has the
  files - seeders don't matter). Direct / free hoster sources
  are exempt (no seed count).
    * New setting: Advanced -> "Filter dead torrents (no/low
      seeders)" (default ON).
    * New setting: Advanced -> "Minimum seeders" slider
      (range 0..50, default 1).
    * Safety net: if the filter would remove every result the
      original list is kept so you never see "no sources" purely
      because seed counts are unknown.
- Fixed: TorBox device-code QR code displayed a blank/dim screen
  because the old implementation layered an OK dialog on top of
  Kodi's full-screen picture viewer, hiding the QR. Replaced with
  a custom split-pane dialog: SMALLER QR on the LEFT, full
  instructions (URL + code) on the RIGHT, both visible at the
  same time. Press OK or Back to close.
