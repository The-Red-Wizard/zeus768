v2.5.0 (2026-04-04)
================================================================================

  PERFORMANCE
  -----------
  * Superfast concurrent scraping - all 35+ scrapers now fire simultaneously
    using ThreadPoolExecutor (20 workers) instead of running one by one
  * 30-second global timeout - slow scrapers get abandoned, fast results arrive
    instantly. Typical scrape time reduced from 60-90s to 5-15s
  * Pre-scrape engine also concurrent (4 workers, 15s timeout)
  * 24/7 Channels also use concurrent scraping (15 workers, 20s timeout)

  BUG FIXES
  ---------
  * Fixed 24/7 Channels completely broken - scrapers were never instantiated
    (called instance methods on classes instead of objects)
  * Fixed 24/7 Channels calling non-existent get_movie_sources()/get_episode_sources()
    methods. Now correctly uses search() like the main scraper engine
  * Fixed pre-scrape engine same instantiation and method bugs
  * Compact donation dialog - single QR + link dialog, no multi-step select menu

v2.1.1 (2026-03-28) - The Big Update
================================================================================

  BUG FIXES
  ---------
  * Fixed rescrape loop - Replaced setResolvedUrl with xbmc.Player().play()
    so Kodi no longer refreshes the container and re-triggers a scrape after
    a movie or episode finishes playing
  * Fixed Trakt API crash - Rewrote trakt_api.py to use native urllib instead
    of the requests module which is not available in a clean Kodi install
  * Fixed TMDB Popular/Trending crash - Same requests-to-urllib migration
    applied to the TMDB module (fixed in v2.1.0, carried forward)
  * Removed repository.gujal from addons.xml (user request to clean up repo)
  * Fixed settings.xml to use Kodi 18+ backward-compatible XML format
  * Cleaned all "emergence" / workspace artifacts from addon files and zips

  FREE STREAMS (NEW - Mobiflix-style)
  ------------------------------------
  * 7 free stream providers that play directly in Kodi - no debrid required:
    - VidSrc.to
    - VidSrc.me (multiple mirror domains)
    - 2Embed
    - AutoEmbed
    - MultiEmbed
    - Embed.su
    - VidLink
  * Uses TMDB IDs to fetch streams (IDs now pass through the entire source chain)
  * Direct m3u8/mp4 URL detection - bypasses ResolveURL for instant playback
  * Free sources highlighted in [COLOR orange] and always sorted first

  AUTOPLAY
  --------
  * Toggle in Settings > General
  * Auto-selects the best available source and plays immediately
  * Prioritizes free direct streams over torrent sources
  * Skips the source selection dialog entirely when enabled

  SOURCE CACHING
  --------------
  * Scraped sources cached in local SQLite database
  * Configurable cache duration (default: 2 hours, range: 1-12)
  * On revisit: "Use Cached / Re-Scrape" prompt (auto-selects cached after 10s)
  * Separate "Clear Source Cache" option in Tools menu
  * Works with pre-emptive scraping for seamless next-episode playback

  FAVORITES / BOOKMARKS
  ---------------------
  * New "Favorites" entry in main menu
  * Separate lists for Movies and TV Shows
  * "Add to Favorites" context menu on every TMDB listing (Popular, Trending,
    Search results, etc.)
  * "Remove from Favorites" context menu in the favorites view
  * Stored locally in SQLite - works without Trakt authorization
  * Saves title, year, TMDB ID, poster art, overview, rating

  UP NEXT (AUTO NEXT EPISODE)
  ---------------------------
  * When a TV episode reaches the last 120 seconds, a dialog prompts:
    "Play next episode? [Show] S01E02" with Play Next / Stop buttons
  * Auto-closes after 30 seconds (defaults to Stop if no input)
  * Works with source caching - if next episode was pre-emptively scraped,
    playback starts instantly
  * Toggle in Settings > Playback

  SKIP INTRO
  ----------
  * During the first 90 seconds of a TV episode, a "Skip Intro?" dialog appears
  * Configurable duration (30-180 seconds) in Settings > Playback
  * Seeks forward to the configured timestamp on "Skip"
  * Auto-closes after 8 seconds if no input
  * Toggle in Settings > Playback

  PRE-EMPTIVE SCRAPING
  --------------------
  * At 75% through a TV episode, SALTS background-scrapes the next episode
  * Results are cached so when Up Next triggers, sources are already available
  * No UI interruption - runs silently in the playback monitor thread
  * Toggle in Settings > Source Cache

  QUALITY PRESETS
  ---------------
  * Tools > Quality Presets
  * 4 built-in presets:
    - Best Quality (WiFi): min 1080p, no CAM, manual source selection
    - Fast Play (Mobile): any quality, autoplay enabled
    - Data Saver: min 480p, autoplay enabled
    - 4K Only: min 4K, no CAM, manual source selection
  * Create custom presets from current settings
  * Delete custom presets via context menu
  * Applies min_quality, filter_cam, auto_play, sort_by, source_timeout

  SCRAPER PRIORITY ORDERING
  -------------------------
  * Tools > Scraper Priority
  * Lists all scrapers with current priority number and ON/OFF status
  * Context menu actions: Move Up, Move Down, Set Priority, Toggle Enable
  * Lower number = higher priority (scrapers run in priority order)
  * Persisted in SQLite - survives addon restarts

  CUSTOM SOURCE DIALOG
  --------------------
  * Replaced plain Kodi directory listing with Dialog().select()
  * Header shows: total sources, free stream count, quality breakdown
    Example: "SALTS: 78 sources (12 free) [4K: 3 | 1080p: 45 | 720p: 20]"
  * Color-coded entries:
    - Orange: Free direct streams
    - Gold: 4K / 2160p
    - Lime: 1080p / HD
    - Cyan: 720p
    - White: SD and below
  * Each entry shows: [Quality] [FREE] [Scraper] Seeds | Size | Title
