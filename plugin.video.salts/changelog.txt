SALTS - Stream All The Sources
Changelog
=========

v2.9.5 (2026) - butterbeansrstop ENHANCEMENTS
- Added: settings for butterbeansrstop scraper in Settings ->
  Streaming Sites:
    * butterbeansrstop_enabled (default: true)
    * butterbeansrstop_username (optional)
    * butterbeansrstop_password (optional, hidden)
    * butterbeansrstop_cloudscraper (default: true)
- Added: cookie-based login session. When username/password are set
  the scraper performs a one-time POST to /index.php and persists the
  session cookies to <profile>/butterbeansrstop_cookies.lwp so later
  requests reuse them. Logged-in requests expose the real iframe URL
  on each embed-selector link where available.
- Added: cloudscraper fallback. If a direct urllib request returns
  403 / 503 (Cloudflare challenge) the scraper retries the same URL
  through cloudscraper (cookies are mirrored back into the jar), so
  users just need `pip install cloudscraper` on the Kodi host - no
  code change required when/if they add it.
- Added: direct-URL capture on the embed-selector tag (href /
  data-link / data-url). When present the source URL is the actual
  embed URL instead of the srstop.link episode page, letting
  ResolveURL handle playback end-to-end.

v2.9.4 (2026) - NEW SCRAPER: butterbeansrstop (srstop.link)
- Added: new scraper module scrapers/butterbeansrstop_scraper.py named
  "butterbeansrstop" which scrapes TV episode streaming sources from
  https://srstop.link.
- Episode page parser reads each <a class="embed-selector"> entry and
  extracts:
    * host       - from the .embed-flag "favicons?domain=HOST" URL
    * quality    - from sibling span classes: vris1080=1080p,
                   vris720=720p, vris480=SD, vrislow=SD
    * url        - the srstop.link episode page itself (the site
                   resolves the real iframe client-side / behind login,
                   so returned sources are non-direct and go through
                   ResolveURL or an external open)
- Resolution flow: try direct slug guesses
  (/show/<slug>-sNNeNN/season/N/episode/N) first, fall back to the
  site's /index.php?menu=search query, finally fall back to parsing a
  show page to recover the real slug.
- Movie search returns an empty list - srstop.link is TV-only.
- Registered in scrapers/__init__.py under ALL_SCRAPERS (Legacy
  Streaming Sites block) and in the "streaming_sites" category list.
- Follows the same BaseScraper interface / caching model as the other
  streaming-site scrapers (watchseries, primewire, movie4k).

v2.9.3 (2026) - PUNCHPLAY BETA NOTICE
- Added a one-shot Kodi toast informing users that PunchPlay integration is
  in beta. Fires the first time the user opens the main menu with PunchPlay
  enabled, or opens the PunchPlay menu directly. Flag is persisted in
  addon_data/punchplay_beta_seen.json so it never re-appears.
- PunchPlay main-menu entry now carries a small "[BETA]" tag.
- Continue Watching row carries a small "[BETA]" tag.
- PunchPlay submenu shows a non-clickable "[BETA] rough edges expected" line
  at the top so users always have context, even after the toast is dismissed.

v2.9.2 (2026) - CONTINUE WATCHING (PunchPlay /api/playback)
- Added a "Continue Watching" row at the top of the SALTS main menu, pulled
  from PunchPlay's /api/playback endpoint (Jellyfin/Plex-style resume bar).
- Ships as a stub: the row is hidden entirely until /api/playback returns
  HTTP 200. Nothing changes for users on PunchPlay builds that don't yet
  expose the endpoint - it will light up automatically once deployed.
- Endpoint availability is cached for 60s so the main menu stays snappy.
- Items show percent-complete in the label and set Kodi's ResumeTime /
  TotalTime properties (progress bar in the default skin). Clicking an
  item routes into the existing get_sources flow with the right TMDB ids
  / season / episode.
- New setting: PunchPlay -> "Show 'Continue Watching' on main menu (when
  API available)" (default: on). Respects punchplay_enabled + login state.
- Only read access is used; no write endpoints are called.

v2.9.1 (2026) - PUNCHPLAY.TV INTEGRATION
- Added PunchPlay.tv as a parallel tracker alongside Trakt (scrobble + mark-watched
  mirror to your PunchPlay account whenever Trakt scrobbling fires).
- Added device-code login flow (Settings -> PunchPlay -> Login to PunchPlay).
- Added top-level "PunchPlay" menu with status / toggle / login / logout.
- New Settings category "PunchPlay":
    * Enable PunchPlay scrobbling
    * Mark watched on PunchPlay (at end of playback)
    * Mirror Trakt "mark watched" actions to PunchPlay
    * Backend URL (advanced, defaults to https://punchplay.tv)
- Only PunchPlay's publicly documented endpoints are used (/api/auth/device/*,
  /api/scrobble/*). Watchlist/lists/calendar etc. continue to come from Trakt.

v2.8.9 (2026-01-20) - BONES & SCRAPER DISPATCH FIX
- Fixed: Bones scraper was silently skipped when no debrid service was
  configured, which was the root cause of many "no sources / everything
  erroring" reports. Bones is now treated as a free scraper alongside
  VidSrc/2Embed/etc. and runs without debrid.
- Fixed: Bones title matching was too loose; short/common queries matched
  almost every entry in the catalog. Matching now requires exact or full
  word-set equality with a year tolerance so remakes/same-name films no
  longer collide (e.g. Saw / Saw X).
- Fixed: Bones year is now parsed from the stream filename and used to
  disambiguate results.
- Fixed: Streamtape resolver now supports all current obfuscation patterns
  and all streamtape mirror domains (streamtape.to, streamtape.net,
  streamtape.cc, streamta.pe, stape.fun, shavetape.cash, etc.).
- Fixed: Scraper dispatcher now recognizes any scraper exposing
  `is_free = True` and is resilient to scrapers using the legacy
  `search(query, media_type)` signature.
- Improved: Scraper-side exceptions are now logged at WARNING instead of
  DEBUG so failures appear in kodi.log for easier diagnosis.
- Added: `bones_enabled` setting so Bones can be toggled from Settings.
