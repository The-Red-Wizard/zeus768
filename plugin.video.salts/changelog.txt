v2.9.18 (2026-04) - SCRAPER RELIABILITY FIXES
- FIXED: Torrentio reliability - added fallback mirrors
  (torrentio.strem.fun + knightcrawler.elfhosted.com) and a
  modern Chrome 120 User-Agent + Accept/Origin headers so
  Cloudflare-protected endpoints stop returning 403.
- FIXED: Comet was returning HTTP 403 because elfhosted now
  rejects the bare /stream path.  Comet now uses the public
  default-config path and falls back to the comet-cf mirror.
- FIXED: MediaFusion 403 - same header treatment as Comet,
  plus a configurable base URL in settings.
- FIXED: PlanetScrapers (1337x / TPB / RARBG / TorrentGalaxy /
  Torrentio backends) - all HTTP requests now send a modern
  Chrome 120 UA + Sec-Fetch headers and accept gzip, which
  bypasses the Cloudflare WAF that was returning 403.
- NEW: All Stremio aggregators (Torrentio, Comet, MediaFusion,
  CyberFlix, Annatar, PeerFlix, EasyNews+) now read their
  base URL from Settings -> Scraper Sources -> Stremio Debrid
  Addons, so you can paste your own configured manifest URL.
- NEW: "Scraper timeout" setting (Settings -> Advanced),
  default 120s (was 60s) - gives slow scrapers a chance to
  finish before being cancelled.
- NEW: "Show all sources at once" - the source dialog now
  always opens after every scraper has finished or hit the
  timeout (no progressive flicker).

- NEW: MediaFusion registered in the main scraper list - community-run

v2.9.16 (2026-04) - SCRAPER RELIABILITY + CACHED BADGES
- FIXED: 16 scrapers (1337x, YTS, EZTV, TPB, TorrentGalaxy,
  LimeTorrents, Torrentz2, RARBG, PrimeWire, Movie4K, WatchSeries,
  SolarMovie, Jackett, Prowlarr, TorrentAPI, Kickass) were crashing
  with "AttributeError: session" at init time - they were ported
  from `requests` but the base class is native urllib.  Added a
  full requests.Session-compatible shim on BaseScraper, so every
  one of them now runs instead of silently disappearing.
- FIXED: PrimeWire (and other Cloudflare sites) no longer dies on
  CERTIFICATE_VERIFY_FAILED - BaseScraper now uses a tolerant SSL
  context (public HTML only, no auth tokens).
- FIXED: 1337x 403 Forbidden - upgraded UA to Chrome 131, added
  full Sec-Fetch/Upgrade-Insecure-Requests headers + automatic
  mirror fallback (1337x.to -> .st -> .ws -> .is -> .gd).
- FIXED: Transparent gzip/deflate decoding - a handful of torrent
  sites always gzip responses; old code returned garbled HTML and
  parsed zero results.
- IMPROVED: Stream-All-The-Sources global timeout 30s -> 60s and
  thread-pool 20 -> 50 workers.  Previously 65 of 153 scrapers were
  abandoned mid-flight (see log: "88/153 completed").  Now all of
  them get a chance to return sources.
- NEW: Debrid-cached torrents now render as a dark-blue "◆ CACHED"
  badge and the whole row is tinted navy blue so instant-play
  torrents jump out at the very top of the source list.  Header
  also highlights the cached count in dark blue.
- HARDENED: Per-scraper HTTP timeout clamped to 5-15s so a single
  slow mirror can't eat the whole 60s budget.

v2.9.9 (2026) - HOTFIX: JSON CACHE CRASH
- Fixed: "TypeError: Object of type BonesScraper is not JSON serializable"
  raised at the end of every Get Sources run. Source dicts emitted by
  Bones, TheChains Wrestling, TheChains Bones Shows and the
  Streamtape/LuluVid scrapers carried a `'class': self` reference that
  blew up json.dumps() inside db_utils.cache_sources. The unused
  'class' field has been removed from all affected scrapers.
- Hardened: db_utils.cache_sources now sanitizes the sources payload
  before json.dumps, dropping any value that isn't a JSON primitive
  (str/int/float/bool/None/list/dict). Future scraper bugs of this
  shape can no longer crash the source cache write.

  data-link / data-url). When present the source URL is the actual
  embed URL instead of the srstop.link episode page, letting

v2.9.4 (2026) - NEW SCRAPER: butterbeansrstop (srstop.link)
- Added: new scraper module scrapers/butterbeansrstop_scraper.py named
  "butterbeansrstop" which scrapes TV episode streaming sources from
  https://srstop.link.
- Episode page parser reads each <a class="embed-selector"> entry and
  extracts:
    * host       - from the .embed-flag "favicons?domain=HOST" URL
    * quality    - from sibling span classes: vris1080=1080p,
                   vris720=720p, vris480=SD, vrislow=SD
    * url        - the srstop.link episode page itself (the site
                   resolves the real iframe client-side / behind login,
                   so returned sources are non-direct and go through
                   ResolveURL or an external open)
- Resolution flow: try direct slug guesses
  (/show/<slug>-sNNeNN/season/N/episode/N) first, fall back to the
  site's /index.php?menu=search query, finally fall back to parsing a
  show page to recover the real slug.
- Movie search returns an empty list - srstop.link is TV-only.
- Registered in scrapers/__init__.py under ALL_SCRAPERS (Legacy
  Streaming Sites block) and in the "streaming_sites" category list.
- Follows the same BaseScraper interface / caching model as the other
  streaming-site scrapers (watchseries, primewire, movie4k).

v2.8.9 (2026-01-20) - BONES & SCRAPER DISPATCH FIX
- Fixed: Bones scraper was silently skipped when no debrid service was
  configured, which was the root cause of many "no sources / everything
  erroring" reports. Bones is now treated as a free scraper alongside
  VidSrc/2Embed/etc. and runs without debrid.
- Fixed: Bones title matching was too loose; short/common queries matched
  almost every entry in the catalog. Matching now requires exact or full
  word-set equality with a year tolerance so remakes/same-name films no
  longer collide (e.g. Saw / Saw X).
- Fixed: Bones year is now parsed from the stream filename and used to
  disambiguate results.
- Fixed: Streamtape resolver now supports all current obfuscation patterns
  and all streamtape mirror domains (streamtape.to, streamtape.net,
  streamtape.cc, streamta.pe, stape.fun, shavetape.cash, etc.).
- Fixed: Scraper dispatcher now recognizes any scraper exposing
  `is_free = True` and is resilient to scrapers using the legacy
  `search(query, media_type)` signature.
- Improved: Scraper-side exceptions are now logged at WARNING instead of
  DEBUG so failures appear in kodi.log for easier diagnosis.
- Added: `bones_enabled` setting so Bones can be toggled from Settings.
