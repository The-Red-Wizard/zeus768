v4.4.0 (2026-02-XX) - AUTH MATRIX + NETWORK SPEED TEST
=======================================================
NEW - AUTHORISATION VIEW:
- New menu: Auth > Authorisation View (Matrix)
- Matrix summary: one line per installed addon showing which
  services (RD / PM / AD / TB / Trakt) are authorised, the
  masked token value, the matching setting id, and whether the
  value came from The Accountant's vault (SYNCED) or was set
  externally by the user (external).
- Account lookup: each service's vault token is resolved via
  live API call to show the username / customer ID / VIP tier
  / days-left for proof of authorisation.
- Drill-down: pick a specific addon to see every service's
  full status + setting id + masked value + resolved account.
- Also reports IPTV provider count from iptv_vault.json.

NEW - NETWORK SPEED TEST:
- New menu: ONE-CLICK SPEED OPTIMIZER > NETWORK SPEED TEST
- Choice of test size: 10 MB / 25 MB / 100 MB
- Live progress with cancel support
- Uses Cloudflare speed.cloudflare.com (Hetzner 100MB fallback)
- Reports: download Mbps, TCP latency to 1.1.1.1, elapsed time,
  friendly rating (4K-ready / 1080p / SD / poor)

FIX - SPEED OPTIMIZER FREEZE:
- Reworked speed_optimizer(): previous version could appear to
  hang for minutes on Kodi installs with large thumbnail trees
  because get_size() walked the whole tree twice (pre + post)
- _fast_size() caps the walk at 5000 entries (approximate
  but bounded)
- Thumbnails pre-measure skipped; we only report how many
  >30-day-old files were pruned
- Cancel checks added between every stage so Back key works
- Final dialog now reports freed MB + thumbnails pruned count

NEW FILES:
- resources/lib/auth_matrix.py   (matrix + drill-down renderer)
- resources/lib/speed_test.py    (Cloudflare/Hetzner downloader)


v4.3.0 (2026-02-XX) - VAULT QR TRANSFER
========================================
NEW - ONE-SCAN DEVICE MIGRATION:
- Export: compress + encrypt your vault locally, upload the
  encrypted blob to Litterbox (1h / 12h / 24h / 72h retention),
  show a QR code pointing to the download URL.
- Import on new Kodi device: scan the QR with your phone, paste
  the URL into Accountant, enter the passphrase - done.
- Crypto: PBKDF2-SHA256 key derivation (120k iterations) +
  HMAC-SHA256 counter-mode stream cipher + authentication tag.
  Uses only Python stdlib - no external deps required.
- Wrong passphrase or tampered payload is detected and rejected
  before any data is touched.

NEW FILE:
- resources/lib/vault_transfer.py

NEW MENU ITEM:
- Auth > Vault QR Transfer
    - Export vault (upload + show QR)
    - Import vault (from URL + passphrase)
    - How it works


v4.2.0 (2026-02-XX) - AUTO-SYNC + EDITABLE SYNC MAP
====================================================
NEW - DAILY AUTO-SYNC ON STARTUP:
- Background service now runs a silent credential sync once
  every 24 hours, on top of the existing expiry checks
- Silently pushes vault creds to every installed addon
  (map-driven + dynamic scanner) with no dialogs
- Shows a single compact notification when done
- New "Auto-Sync Settings" menu (under Auth):
    - Enable / Disable daily auto-sync
    - Run auto-sync now
    - See last sync timestamp
- Controlled by vault flag 'auto_sync_enabled' (default on)
- Honors 'last_auto_sync' throttle stored in vault

NEW - USER-EDITABLE SYNC MAP:
- ADDON_SYNC_MAP is now stored as JSON
- Default bundled at: resources/data/sync_map.json
- User copy at: special://profile/addon_data/
                  plugin.program.theaccountant/sync_map.json
- Auto-copied on first run, never overwritten unless user resets
- New "Sync Map Manager" menu (under Auth):
    - View current mappings
    - Reload after external edit (no Kodi restart needed)
    - Reset to bundled defaults
    - Help dialog with supported service/vault keys
- Python ADDON_SYNC_MAP is kept as a legacy fallback only

REFACTORING:
- auth_manager.sync_to_all_addons() now takes silent=True for
  background use (suppresses progress dialogs and popups)
- service.py refactored: unified loop tick (1h) with internal
  throttling, shared vault load/save helpers
