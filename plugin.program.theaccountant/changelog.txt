v4.7.0
- Trakt card now flexes your taste:
  * TOTAL WATCH TIME - pulled from the
    /users/<user>/stats endpoint
    (movies.minutes + episodes.minutes).
    Auto formatted: shows "Xh watched" for
    anything under a day, switches to
    "X.Xd watched" for longer totals.
  * TOP GENRE - tallied from
    /users/<user>/watched/shows?extended=full.
    Every genre on every watched show is
    counted, weighted by the show's play count,
    so your most-binged genre wins. Shown
    title-cased (e.g. "Science Fiction").
- Both values appear in the status line of the
  Trakt card, alongside VIP/Standard, username
  and joined date.
- Purely additive - existing detail row
  (watched / collected / watchlist / lists) is
  unchanged.

v4.6.8

  AttributeError: module 'resources.lib.auth_manager'
  has no attribute 'get_rd_account_info'
  when Real-Debrid was authorized. The function body
  existed but its `def get_rd_account_info(token):`
  line had been dropped in 4.6.7 so Python never
  exposed it. Restored the definition - RD now
  appears correctly on the Account Cards window.
- FIX: "Sync All to Addons" (Auto / Per-service /
  TMDB Helper sync / IPTV export) no longer causes
  Kodi to pop the "Install addon?" / dependency
  prompt for addons that aren't part of the user's
  setup. The code now pre-checks with
  System.HasAddon(<id>) and silently skips any
  addon that isn't already installed, instead of
  blindly calling xbmcaddon.Addon(id) which on
  Kodi 20/21 (Omega) can trigger the installer UI.
- Result: syncing debrid / Trakt / TMDB / IPTV
  credentials only writes to addons you already
  have, exactly as intended.


v4.6.7
- Speed Test FIX: overlapping text + non-moving needle
- Layout:
  * Gauge is now a proper speedometer: the arc opens downward
    with 0 at the lower-left and max at the lower-right, sweeping
    across the TOP. This frees the area under the gauge center
    for the readouts so nothing overlaps.
  * Status line, separator, LATENCY/DOWNLOAD/UPLOAD stats row
    and the START NEW TEST button are now spaced out vertically
    with ~40 px gaps between them.
  * Removed the redundant "Press ENTER / BACK" hint line; the
    button label itself now says "START NEW TEST (press ENTER)"
    and BACK always closes.
- Needle:
  * Gauge max lowered from 1000 to 500 Mbps (covers the vast
    majority of home connections) so the needle sweeps a much
    more visible angle for typical speeds (e.g. 100 Mbps moves
    the needle 54 degrees instead of 27).
  * Needle segments are now created ONCE and repositioned on
    every update via setPosition() instead of being deleted and
    recreated every frame. This removes flicker and ensures
    the needle keeps up with the download loop.
  * UI updates during the download loop are throttled to ~15
    fps so the needle animates smoothly without pegging CPU.
- Download test now requests a 100 MB stream from Cloudflare
  (was 10 MB) for a more accurate measurement on fast lines.
- Upload test uses a 2 MB payload (was 500 KB).
- Numeric speed readout uses one decimal below 100 Mbps and
  integer above, so slow connections still show movement.

- FIX: Account Cards window and Speed Test speedometer were
  rendering on a fully transparent background because the
  ControlImage objects used an empty texture path with only
  colorDiffuse. Kodi's ControlImage needs an actual texture

  drawn and the fanart/skin shows straight through, making
  the UI very hard to read.
- Added resources/media/white.png (a 4x4 solid-white PNG)
  and used it as the base texture for every background,
  panel, row, border, gauge segment and needle segment in
  both account_cards_window.py and speed_test_window.py.
- Account Cards rows are now a solid dark-blue
  (FF13304f) instead of a 30% white wash, so the red
  "Not authorized" / green "Authorized" status text is
  fully legible over it.
- No functional changes; purely a visual readability fix.
