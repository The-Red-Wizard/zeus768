v3.7.13 (May 2026)
================================================================================
  * NEW:    Auto-play next episode with 5-second countdown (configurable)
              - Countdown starts 1 minute before episode ends (during credits)
              - Enabled by default, configurable in Settings > Playback
              - Countdown duration: 5, 10, 15, 20, or 30 seconds
              - Automatically detects next episode in season or next season
              - User can cancel with Back button during countdown
  * NEW:    Intro skipping functionality
              - Uses TMDB episode data for intelligent intro detection
              - Two modes: Manual (notification) or Auto-skip
              - Heuristic detection: 45s (sitcoms), 75s (dramas), 90s (long-form)
              - Fallback duration configurable: 30-120 seconds
              - Enabled by default, manual mode
              - Settings: Enable intro skipping, Auto-skip intros, Fallback duration
  * IMPROVED: Up Next monitoring now respects user settings for countdown
  * IMPROVED: Next episode detection includes cross-season support


v3.7.12 (January 2026)
================================================================================
  * FIX:   Playback no longer re-resolves the link or briefly flashes
           "Playback failed" before finally starting. The user-visible
           symptom was:
              "Resolving with Real-Debrid..."   (1st time)
                  -> "Playback failed"  dialog
                  -> "Resolving with Real-Debrid..."  (2nd time)
                  -> stream finally plays
           Root cause: play_source() was calling xbmc.Player().play()
           directly AND then signalling the plugin done with
           setResolvedUrl(HANDLE, False, ...). The False flag tells
           Kodi "I failed to resolve" -> Kodi shows the Playback failed
           dialog AND re-invokes the same plugin URL, which kicks off
           a second debrid resolution.
              - play_source() now hands the resolved stream URL to
                Kodi via setResolvedUrl(HANDLE, True, ListItem(path=...))
                whenever the function is invoked from a plugin URL
                (HANDLE >= 0). Kodi takes over playback - no second
                resolution, no failure dialog, no replay loop.
              - When called outside a plugin URL context (e.g. directly
                from a custom dialog without an active handle),
                xbmc.Player().play() is still used as a fallback.
              - ListItem.setContentLookup(False) is now applied so Kodi
                doesn't fire a HEAD probe at the debrid hotlink before
                playback (some Real-Debrid hosts 403 the probe and
                abort playback as a side-effect).
              - Resume-point seeking and the Up Next / Trakt / PunchPlay
                episode monitor still attach to xbmc.Player() once Kodi
                starts the stream, so all of that behaviour is preserved.


v3.7.11 (January 2026)
================================================================================
  * FIX:   "Resolving with Real-Debrid..." (and the Premiumize / AllDebrid /
           TorBox variants) progress dialog could stay visible on top of the
           player after the stream had already started playing.
              - play_source() now uses a non-modal background progress
                dialog (xbmcgui.DialogProgressBG) instead of the modal
                xbmcgui.DialogProgress. Modal dialogs can sit on top of /
                behind the video window when playback is kicked off with
                xbmc.Player() directly from the plugin handler; the
                background progress lives in the bottom-right corner and
                dismisses cleanly the instant the resolver returns.
              - Added a tiny _BGProgress adapter so resolver.py and
                debrid.py keep calling .create / .update / .close / 
                .iscanceled exactly as they did before - no changes
                needed in any of the per-service resolvers.

v3.7.10 (January 2026)
================================================================================
  * FIX:   "Resolving link via debrid..." / "Waiting for TorBox..." dialog
           no longer stays on top of the video once playback starts.
              - play_source() now calls xbmcplugin.setResolvedUrl(HANDLE, False, ...)
                after handing the stream to xbmc.Player so Kodi stops showing the
                bottom-right "Working..." spinner.
              - All residual progress / busy dialogs are explicitly dismissed
                via Dialog.Close(progressdialog|progressdialogbg|busydialog|
                busydialognocancel, true) right before and right after
                Player.play().
              - TorBox.resolve_magnet() no longer re-issues progress.update()
                with the same "Waiting for TorBox..." text every 2 s. The
                redundant updates were keeping the DialogProgress window
                alive across the close() boundary on some Kodi builds, which
                is why the dialog could be seen overlapping the MPAA intro
                / first frames of the stream.
              - Right before unrestrict_link() returns the final URL, the
                dialog text is switched to "Starting playback..." so the
                last visible state isn't a "Waiting..." label.
              - Error and "no source" paths in play_source() now also
                signal plugin completion so the "Working..." indicator
                always disappears.


v3.7.9 (January 2026)
================================================================================
  * NEW:   TorBox authorization now supports BOTH methods.
              - Choose at pair-time: "Device Code (recommended)" or "API Key".
              - Device Code flow uses the official TorBox endpoints
                (/v1/api/user/auth/device/start +
                 /v1/api/user/auth/device/token) and shows a cancellable
                progress dialog while it polls. Includes an optional QR code
                of the verification URL (torbox.app/oauth/device) so you can
                sign in from your phone — no typing on the remote.
              - API Key flow is unchanged: paste, validate, done.
              - Settings &gt; TorBox now has a separate "Show QR Code" entry
                matching the other debrid services.
  * No functional regressions vs 3.7.8.

v3.7.8 (January 2026)
================================================================================
  * NEW:   TorBox debrid is now first-class.
              - Authorization is a single API key entry (Settings > Debrid >
                Authorize TorBox). No more device-code wait.
              - Magnet resolution rewritten: form-encoded createtorrent,
                cached/queued state polling, video-only file selection with
                sample/extras filtering, picks the largest playable file,
                cleans the torrent up after a one-shot resolve.
              - Cache checks now use the official POST /torrents/checkcached
                endpoint with a JSON hash list (single + batch).
              - Cleaner failure modes: invalid/expired keys are detected on
                pair, dead torrents are removed instead of left hanging.
  * UI:    Netflix-style skin is now OFF by default for new installs.
              - Existing users keep their current setting.
              - Toggle it from Settings > Appearance > "Enable Netflix-Style
                Skin" whenever you want the immersive look back.
  * No functional regressions vs 3.7.7.
